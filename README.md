# IOI in GPT-2 small — full hidden-dimension attribution patching

One self-contained notebook that looks at **all hidden-dim activations** of
GPT-2 small on the **IOI** task (not just hand-picked MLP neurons / heads /
the residual stream), using **attribution patching**, then:

1. **recovers** the IOI circuit,
2. **traces** the paper's circuit (Fig 2) annotated with the new numbers,
3. **regenerates the paper's figures** (Fig 1, 2, 3b, 4b, 5b, 7, 8) from our attribution,
4. looks for **circuits beyond the paper**.

Dataset: [`mib-bench/ioi`](https://huggingface.co/datasets/mib-bench/ioi).
Paper: *Interpretability in the Wild* (Wang et al. 2022, arXiv:2211.00593) — in `paper/`.

## Run

```bash
pip install torch transformers datasets huggingface_hub numpy matplotlib jupyterlab
jupyter lab ioi_full_activations.ipynb     # runs end-to-end on a GPU (~minutes, 1000 examples)
```

Everything (data, attribution engine, head + edge attribution patching, all the
figures) is inline in the notebook. Generated plots are saved to `figures/`.

## Key results (1000 examples, ~3.5 clean logit diff vs paper's 3.56)

- 19/26 circuit heads recovered in the top-26; per-position maps recover the
  positional structure (name movers/S-inhibition @END, induction @S2).
- Fig 3b: name movers 9.9/9.6/10.0 (+), negative name movers 10.7/11.10 (−).
- Fig 4b (effect on NM queries) lights up the S-inhibition heads 7.3/7.9/8.6/8.10.
- Fig 5b (effect on SI values) lights up induction 5.5/6.9 and duplicate 3.0.
- Beyond the paper: head **9.4** behaves like an unlisted negative name mover;
  a sparse set of MLP neurons matters (paper doesn't analyse MLPs); the choice of
  counterfactual decides which sub-circuit is even visible.

Attribution / edge-attribution patching are first-order approximations — great
for ranking and maps; confirm shortlisted nodes with exact patching.
